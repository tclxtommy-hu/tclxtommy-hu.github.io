# RAG 技术指南

检索增强生成（RAG，Retrieval-Augmented Generation）是一种将「检索」与「生成」结合的
大语言模型应用范式。它让模型在回答问题时，先从一个外部知识库中检索相关资料，
再基于检索到的资料生成答案。

## 核心流程

1. 用户提出问题；
2. 系统把问题向量化，在向量库中检索最相关的若干片段；
3. 将这些片段作为上下文拼进提示词；
4. 大语言模型基于上下文生成最终回答。

## 为什么需要 RAG

- 缓解大模型幻觉：答案有据可依，而非凭空生成；
- 引入私有/最新知识：无需微调即可使用专属数据；
- 可解释、可追溯：回答能对应到具体来源片段。

## 关键组件

- 文档加载与清洗；
- 文本分块（Chunking）；
- Embedding 向量化；
- 向量存储与检索；
- 提示词编排与生成。
