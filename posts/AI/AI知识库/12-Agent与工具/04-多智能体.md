# 多智能体

> 一句话定义：多个分工不同的 Agent 协作完成复杂任务，由编排者协调流程。

## 1. 定义
Multi-Agent 系统由多个角色不同的 Agent（规划者、研究员、编码者、评审者…）协作，通过消息传递/共享状态协同，由 Orchestrator 编排。

## 2. 编排模式
- **Hub-and-Spoke**：中心分发汇总。
- **Pipeline**：流水线串行（研究→写作→评审）。
- **Hierarchical**：分层委派。
- **Debate/Voting**：多 Agent 辩论投票。

## 3. 优势
- 处理单 Agent 难胜任的复杂任务。
- 每个 Agent 专精一面，质量更高。
- 独立子任务可并行。
- 评审/辩论 Agent 提升可靠性。

## 4. 劣势
- 成本与延迟数倍于单 Agent。
- 协调复杂，易死锁/活锁。
- 调试困难。
- 简单任务上过度工程。

## 5. 注意事项
- 先单 Agent，不够再上多 Agent。
- 角色边界清晰，避免抢活。
- 明确终止条件防无限辩论。
- 设总预算上限。
- 人在环上关键节点确认。

## 6. 代表框架
- AutoGen（微软）、CrewAI、MetaGPT、LangGraph 多 Agent。

## 7. 学习要点
- 多 Agent 是单 Agent 的规模化协作扩展。
- 成本量级远高于单 Agent，慎用。
- 角色分工 + 编排模式是设计核心。
