# 项目全局链接规则（ALL POSTS 通用）

> 位置：`posts/.build-link-guide.md`
> 适用范围：`posts/` 下**所有**目录的 markdown 文件
> 关键脚本：`scripts/build-posts.js`

---

## 一、你要记住的唯一规则

**这个项目的构建系统把 markdown 文件分成两类，链接写法不同：**

```
┌────────────────────┬───────────────────────────────────────┐
│      文件类型       │          链接该用什么扩展名？           │
├────────────────────┼───────────────────────────────────────┤
│ README.md          │  .md    ← rewriteReadmeLinks 自动转换   │
│ 其他所有 .md 文件   │  .html  ← marked 渲染后保持原样，不转换  │
└────────────────────┴───────────────────────────────────────┘
```

> **一句话口诀：README 用 .md，其他用 .html**

---

## 二、为什么？——构建流水线原理

```
posts/xxx/README.md
    │
    ├──→ 被 build-posts.js 跳过，不生成独立 HTML 页面  ←── 关键！
    │
    └──→ 内容被提取到 archive.html 内联展示
         └──→ 此时 rewriteReadmeLinks() 把文中 .md 链接都转成 /posts-html/.../...html

posts/xxx/任意非README名.md
    │
    ├──→ marked 直接转 HTML → 生成独立页面 /posts/xxx/任意名.html
    │
    └──→ 链接保持原样，不做任何转换  ←── 关键！
         写 .md → 页面上就是 .md → 404
         写 .html → 页面上就是 .html → 正确跳转
```

---

## 三、实操决策表

新建或修改 markdown 文件时，按此表决定链接写法：

| 我当前在编辑的文件是 | 链接到同目录其他 post | 链接到子目录 post | 链接到上级/同级目录 |
|---------------------|---------------------|------------------|-------------------|
| **README.md** | `[文字](文件名.md)` | `[文字](子目录/文件.md)` | `[文字](../上级/文件.md)` |
| **普通 .md post** | `[文字](文件名.html)` | `[文字](子目录/文件.html)` | `[文字](../上级/文件.html)` |

---

## 四、每次新建 post 目录时的检查清单

当你创建新目录并往里面放 md 文件时，逐个检查：

- [ ] 这个文件的名称是不是 `README.md`？
  - **是** → 内部所有指向其他 md 的链接，用 `.md` 扩展名
  - **否** → 内部所有指向其他 md 的链接，用 `.html` 扩展名
- [ ] 如果我写了 README.md，它指向了其他 README.md 吗？
  - 允许，但被指向的 README.md 不会生成独立页面（构建脚本第31行跳过）
- [ ] 链接的相对路径是否正确？
  - 从当前文件所在目录出发计算相对路径
  - 可以安全使用 `../` 上溯

## 五、反面教材（会 404 的写法）

```markdown
<!-- ❌ 普通 post 里用了 .md 链接 — marked 不转换，直接渲染成 .md URL，404 -->
| 01 | [ReAct](01-ReAct模式.md) |

<!-- ❌ README 里用了 .html 链接 — rewriteReadmeLinks 只匹配 .md 结尾的链接 -->
| 01 | [ReAct](01-ReAct模式.html) |
```

## 正面教材（正确的写法）

```markdown
<!-- ✅ 普通 post（非README）：用 .html -->
| 01 | [ReAct](01-ReAct模式.html) |

<!-- ✅ README.md：用 .md -->
| 01 | [ReAct](01-ReAct模式.md) |

<!-- ✅ README.md 跨目录引用：用 ../ + .md -->
| 上游 | [总索引](../README.md) |
```

## 六、源码参考

| 行号 | 作用 |
|------|------|
